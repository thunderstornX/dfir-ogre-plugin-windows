## Contributing

We actively encourage and accept a wide range of contributions, including :  
- Security enhancements and hardening  
- Bug fixes and performance improvements  
- Code cleanups, refactoring, and maintainability upgrades  
- Documentation and example updates  
- New features (subject to alignment with the project’s vision)  

## Support Policy  
We aim to give constructive feedback promptly, but response times may vary depending on our availability. 

* Assistance is provided primarily to contributors working on pull requests.  
* Feature requests and non-security-related bug reports are welcome but will be prioritized according to project needs.
* General usage help is done on a best-effort basis.

## Reporting Vulnerabilities  
If you discover a security vulnerability, please report it confidentially following the guidelines in [SECURITY.md](SECURITY.md).
